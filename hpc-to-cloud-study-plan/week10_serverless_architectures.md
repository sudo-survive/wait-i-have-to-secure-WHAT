# Week 10: Serverless Architectures
## From Batch Jobs to Event-Driven Functions

Complete week covering Lambda functions, event sources, Step Functions workflows, and serverless security best practices for scientific computing workloads.

---

## Learning Objectives

- Understand serverless computing vs. HPC batch processing
- Implement AWS Lambda, Azure Functions, and Google Cloud Functions
- Design event-driven architectures for scientific data processing
- Create workflow orchestration with Step Functions
- Apply serverless security and cost optimization
- Migrate HPC batch jobs to serverless functions

---

## Key Topics

### Day 1: Serverless Fundamentals
- Lambda functions vs. HPC jobs
- Event sources and triggers
- Function configuration and limits
- Cold starts and performance optimization

### Day 2: Event-Driven Architectures
- S3 events for data processing
- SQS/SNS for messaging
- EventBridge for event routing
- API Gateway integration

### Day 3: Workflow Orchestration
- AWS Step Functions
- State machines for scientific workflows
- Error handling and retries
- Parallel processing patterns

### Day 4: Serverless Security
- Function IAM roles and permissions
- Secrets management
- VPC configuration
- Security monitoring and logging

### Day 5: Cost Optimization
- Function sizing and optimization
- Reserved concurrency
- Cost monitoring and alerts
- Serverless vs. traditional cost comparison

---

## Deliverable

Serverless architecture design for scientific data processing workflows with security controls, cost optimization, and migration plan from HPC batch jobs.
